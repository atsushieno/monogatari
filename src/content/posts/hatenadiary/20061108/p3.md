Monoに足りないASP.NET 2.0の部品に新しいコンパイルモデルであるApp\_Codeのサポートというものがあるのだけど、コレを実装しているハカーがいたらしい。というか、既に動いているらしい。cool!

実際にはMonoの場合はweb.configに[CodeProviderを追加](http://pastebin.ca/243383)する必要がある。

BuildProviderが次のターゲットだということで、楽しみである。何より、これをやっつけているのがXimian monkeyではないコミュニティの人だっていうのがイイな。

どんどんテストしてほしい、と言っていたので、何か手持ちのアプリケーションがある人は試してやって下さい。

#  
# Alias For mod\_mono Demo Applicatio  
#  
<Location /mono>  
 SetHandler mono-ctrl  
 Order deny,allow  
 Allow from All  
</Location>  
  
MonoServerPath /usr/local/bin/mod-mono-server2  
MonoSetEnv LANG=ja\_JP.UTF-8  
MonoAutoApplication disabled  
MonoRunXSP True  
MonoPath "/usr/lib/mono/2.0"  
MonoDocumentRootDir "/home/hirai/workspace/\_util\_mono"  
Alias /\_util\_mono "/home/hirai/workspace/\_util\_mono"  
MonoApplications default "/\_util\_mono:/home/hirai/workspace/\_util\_mono"  
<Location /\_util\_mono>  
 SetHandler mono  
</Location>  
Include /etc/httpd/conf.d/mod\_mono.conf
